# Debug-практики: Найди и исправь проблему

Этот модуль содержит **сломанный многопоточный код**. Ваша задача — найти баги и исправить их.

## Отличие от обычных практик

В обычных практиках вы пишете код с нуля. Здесь вы получаете **готовый код с багами**, которые нужно найти и исправить. Это более приближено к реальной работе и собеседованиям.

## Структура

Каждый класс содержит один или несколько багов:

| Класс | Тип проблемы | Сложность |
|-------|--------------|-----------|
| `BrokenCounter` | Race Condition | Easy |
| `BrokenVisibility` | Visibility Problem | Easy |
| `BrokenSingleton` | DCL без volatile | Medium |
| `BrokenCache` | Race Condition в check-then-act | Medium |
| `BrokenExecutor` | Утечка потоков | Medium |
| `BrokenDeadlock` | Deadlock | Hard |

## Как работать

1. Прочитайте код в `src/main/java/com/multithreading/debug/`
2. Найдите проблему (баг описан в комментариях к классу)
3. Исправьте код
4. Запустите тесты: `mvn test`
5. Все тесты должны проходить

## Подсказки

- Каждый класс содержит комментарий `// BUG:` рядом с проблемным местом
- Тесты специально написаны так, чтобы с высокой вероятностью обнаружить баг
- Если тест иногда проходит, а иногда падает — это признак race condition

## Типичные ошибки, которые нужно искать

1. **Race Condition**: Операции `read-modify-write` без синхронизации
2. **Visibility**: Изменения в одном потоке не видны другому
3. **Deadlock**: Взаимная блокировка потоков
4. **Check-then-act**: Проверка и действие не атомарны
5. **Resource leak**: Потоки/ExecutorService не закрываются

## Запуск тестов

```bash
mvn test
```

Успех — когда все 6 тестов проходят!
